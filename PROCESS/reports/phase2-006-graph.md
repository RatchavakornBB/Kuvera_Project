Noted in state.md's environment notes so a future session doesn't mistake "no checkpoint tables after db reset" for a bug. `DATABASE_URL` (direct Postgres connection, for the checkpointer only) is a new required config key distinct from `SUPABASE_URL`/`SUPABASE_KEY` (the PostgREST/Storage/Auth API) — both point at the same local Postgres instance but through different protocols (raw Postgres wire protocol vs. HTTP API), so both are genuinely needed, not redundant.
